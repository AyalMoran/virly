Virly Realtime - Playground

A static index for the Socket.IO real-time channel added on this branch. The channel is authenticated with the existing virly_auth JWT cookie, uses per-user rooms, and is proven end-to-end with one event: when a transfer commits, the recipient's client receives transfer:received and silently refreshes its balance and transactions instead of polling.

End-to-end flow

transfer commits -> executeTransfer emits (post-commit, best-effort) -> recipient room user:<id> -> client dispatchRealtimeEvent -> dashboard silent refetch

The channel at a glance

  • Auth - the handshake verifies the same virly_auth httpOnly cookie as the REST API (jwt.verify(token, config.jwtSecret)); unauthenticated sockets are rejected.
  • Rooms - each socket joins user:<userId>, derived server-side from the verified id, so a user only receives its own events.
  • Gateway seam - services emit via getRealtime().emitToUser(userId, event, payload); the default is a no-op so services and the test suite run without a socket server (DI at boot, mirroring getRepositories()).
  • Event contract - transfer:received with payload { amount: number; reason: string | null }.
  • Best-effort - the emit is post-commit and wrapped in try/catch; a realtime failure never breaks a transfer.
  • Scope - single instance; horizontal scaling needs the Socket.IO Redis adapter (deferred).
