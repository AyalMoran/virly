Verification-Tokens Collection Split - Playground

A static index for the work on this branch: email-verification tokens move off the User record into a dedicated verification_tokens collection (Mongo) / table (Postgres), behind a new VerificationTokenRepository in the existing repository seam.

What changed

  • New repository - VerificationTokenRepository (upsertForUser / findByUserId / deleteForUser / deleteExpired), Mongo + Postgres impls behind the seam.
  • Mongo store - VerificationToken model with a TTL index on expiresAt (auto-expiry).
  • Postgres store - verification_tokens table (unique on user_id), migration 0002; swept by the Postgres TTL sweeper.
  • auth.service - issues / verifies / clears tokens via the new store instead of inline User fields.
  • Inline fields removed - verificationTokenHash / verificationTokenExpiresAt dropped from User by migration 0003, which backfills the Postgres data then drops the columns atomically. Mongo backfill is a separate, non-destructive script.
