Notable findings baked into the plans

The "startup throttle" is a global 6-second delay on every request

server/.env (gitignored) sets VIRLY_THROTTLE_MS=6000, and a dev latency simulator middleware in server/src/app.ts applies it to EVERY response, not just boot. The plan reproduces it E2E, extracts a production-guarded throttle module with a loud boot warning, and removes the stale value.

The client is typechecked nowhere in CI

Client Jest runs through @swc/jest, which strips types without checking them; only tsc -b inside the client build checks them, and CI never builds the client. The CI plan closes this with a build job.

Counterparty summaries reach users as prose only

buildBlocksFromResult in server/src/ai/v2/blocks.ts has no case for getCounterpartySummary, so the tool's structured totals are flattened into a sentence. The bento-card plan adds the missing block type end to end.
