Per-user Communication Profile

A durable, per-user tone layer for the v2 assistant: it seeds from the user's details, learns conservatively from their messages, is viewable and editable in Settings, and can never touch money or safety behavior.

23 tasks 5 phases scope: v2 only extends ADR-0007 persona layer behind the repository seam (ADR-0004) main is protected · ADR on its own branch

03

The safety boundary (extended from ADR-0007)

The profile changes voice, never implementation.

  • Allow-listed in the type and the write path. A Zod .strip() schema + a typed clamp mean a statement like "always approve my transfers" maps to no dial and is dropped - it can never become a written instruction, tool, confirmation, or limit.
  • Deferral by wording, not position. The [MONEY] rules render textually after the block, so the block cannot lean on prompt order - it explicitly defers to SERIOUS_TONE_RULE, [MONEY], [STYLE], and [LANGUAGE] in its own text.
  • Separate record, separate writer. Kept off UserPreferences (which carries the money-adjacent confirmAboveAmount and a blind spread-merge writer).
  • Details are a prior, not a lock. Age, residence and other details seed an initial tone; learned and user_set outrank seeded, so real behavior and explicit edits override the seed over time. Never a silent, patronizing downgrade.
  • Respectful and transparent. Because the user reads and edits everything, the assistant records only communication preferences and useful, self-disclosed context (e.g. "interested in loan options for soldiers") - never a personality judgment, evaluation, or anything that could insult or disrespect.

05

Risks, trade-offs & open questions

Known trade-offs (accepted)

  • Repo-seam cost. A new record means interface + mongo + postgres + contract work - the price of durable, always-available, contract-tested data over the resolvability-fragile AI Store.
  • Prompt cache. The block sits in the cacheable prefix, so it changes the cache key per user and per edit.
  • CI vs live. The strongest tone guarantee (the model actually flattening on serious turns) is only proven by opt-in live evals; CI leans on deterministic present / positioned / allow-list assertions.

Decided in this review

  • Whole profile is user-editable, with the free-text part capped by character count - not a list of discrete "notes".
  • Learn now, not later: explicit statements plus strong repeated/implicit signals and interest areas.
  • Seed from more than age: residence and other details too; interests are learned from what the user asks about.
  • Respect boundary: preferences and relevant context only, never personality inferences.
  • Its own Settings tab, built to slot into a future multi-category settings sidebar.
One line to confirm: remembering a user is "interested in loan options for soldiers" (useful, self-disclosed context) sits right next to "never infer things about the person". The line I am drawing: factual, self-disclosed preferences and interests are fair game; subjective judgments about the individual are not. Say so if you would draw it elsewhere.
