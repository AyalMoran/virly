Counterparty hover card

Todoist 6h24RpJcM2XCJH2M: hovering or keyboard-focusing a counterparty email link pops a small card - display name, net balance, you-sent / you-received, transaction count - fetched lazily and cached, with a "View full profile" link.

client / React 18 Radix HoverCard lazy + cached fetch a11y: hover + focus + Escape

Design

A CounterpartyLink wraps the existing <Link> in a Radix HoverCard. On open it lazily fetches GET /api/users/:email/profile (the same endpoint the full profile page uses - no backend change) through a memoized module cache so repeat hovers don't refetch. The card body is a pure, hook-free UserHoverCardContent (unit-testable in the renderToStaticMarkup harness), driven by a pure summarizeRelationship helper. TransactionList swaps its inline counterparty link for CounterpartyLink.

What landed

  • Pure summarizeRelationship helper (net sent/received/even) + tests.
  • Pure UserHoverCardContent (loading / error / loaded) + tests.
  • Memoized fetchUserProfileCached (dedupes per email, doesn't cache failures) + test.
  • CounterpartyLink Radix wrapper + Storybook story (in __stories__/, @storybook/react-vite CSF).
  • Adopted in TransactionList; the row's closest("a") guard keeps the trigger from firing row selection.
  • Dependency added: @radix-ui/react-hover-card.
