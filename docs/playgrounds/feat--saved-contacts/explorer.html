Saved contacts + recent recipients

Todoist 6gfGpV4GVwGxhjPM ("להוסיף אנשי קשר ו-recent"): a persistent contact book alongside the existing transaction-derived "recent payees", surfaced on the transfer page as one recipient book.

full repository seam both drivers contract-tested client + server

Why two different things

  • Recent stays derived (no persistence, self-maintaining) - it already worked.
  • Saved is an explicit user action with a server-side record, because "recent" forgets anyone who drops out of the last 10 transactions. Contact emails must belong to registered users; delete is by id scoped to the owner; upsertForOwner makes saving idempotent.

What landed (6 tasks)

  • Repository seam: ContactRepository interface + Mongoose model + Drizzle table + BOTH driver implementations, wired into both factories, with dual-driver contract tests.
  • Service: ownership + registered-user + no-self checks.
  • HTTP: GET/POST /api/contacts, DELETE /api/contacts/:id + OpenAPI.
  • Client: api methods + a pure mergeRecipientBook(saved, recent) helper (dedupes recents already saved).
  • UI: a hook-free RecipientBook (saved chips with a remove action, recent chips with a save action) + story, adopted on the transfer page.

Verification

  • TDD across all layers. Server unit 1807 pass, client 646 pass; server tsc + client tsc -b clean.
  • Contract: [postgres] contacts 4/4 (the CI-gated driver) + all pre-existing postgres suites green.

Note on [mongo] contract: it could not be run locally - the host's dockerized mongo replica set crashes on file-descriptor exhaustion ("Too many open files"), which times out mongoose.connect() in the harness beforeAll and equally kills every mongo suite, not just contacts. CI does not run the mongo contract anyway (it self-skips without CONTRACT_MONGO_URL; ci.yml sets only the Postgres vars). The mongo contact repository mirrors the existing verified mongo repos and the passing Postgres implementation.

Note on migration: 0005_contacts.sql was hand-written (local drizzle-kit generate hit a drizzle-orm version mismatch); the contract harness's runPgMigrations() applies it cleanly, proving it correct.
