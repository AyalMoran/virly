Root cause

getTransactionsWithCounterparty called the repo with limit: 5 and never populated metadata.transactions - so the v2 block builder rendered no card (prose only). Every caller (v1 tool, v2 getCounterpartyTransactions, v2 searchTransactions-with-counterparty) funnels through this one executor, so both defects were fixed in one place.

The fix

  • transactionHelpers.ts: MAX_TRANSACTION_LIST_LIMIT=50, a messageRequestsAllTransactions predicate (EN "all/every/entire" + HE "הכל/כל ה", excluding "completed"), and getTransactionLimitAllowingAll - resolves the row count from the user's own words.
  • getTransactionsWithCounterparty.ts: uses the resolver (default 10, "all" -> 50, explicit N honored) and populates metadata.transactions via metadataFromTransactionRows so a transaction_list card renders every row, emails masked.
