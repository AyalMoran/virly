Scheduled idempotent RAG sync

Todoist 6h25Jf6p4jfmqF66: run the existing rag:sync --source=drive automatically every 6 hours from inside the server, skip overlapping runs, and email an alert on failure. Opt-in, off by default.

server only no --force advisory-lock guarded opt-in

Design

  • Reusable runner: the orchestration was pulled out of the one-off CLI into src/ai/rag/sync-runner.ts, so the scheduler calls it directly (no shelling out to npm run). The CLI now just parses args and calls the runner.
  • Overlap guard: a session-level Postgres advisory lock (in db/vector.ts) means "skip if a sync is already running" - and unlike an in-process boolean, it also protects against a second server instance or a manual npm run rag:sync colliding with the scheduled run. No new dependency.
  • Scheduler: src/ai/rag/sync-scheduler.ts fires every VIRLY_RAG_SYNC_INTERVAL_MS (default 6h, floor 5min), always --source=drive, never --force.
  • Alerting: a failed run emails VIRLY_RAG_SYNC_ALERT_EMAIL via Resend (falls back to logs if unset).

Config (all opt-in, off by default)

  • VIRLY_RAG_SYNC_ENABLED=false
  • VIRLY_RAG_SYNC_INTERVAL_MS=21600000 (6h; floor 5min)
  • VIRLY_RAG_SYNC_ALERT_EMAIL=
